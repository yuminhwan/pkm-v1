# /일일회고

매일 아침 자동으로 굴러가는 회고 루틴. 두 가지를 한 호출에 처리한다:

1. **`.ai-wiki/index.md` 동기화** — outbox 본문 변경사항 반영 + 메타인지·열린 질문 갱신
2. **"오늘 한 줄" 가이드 출력** — 누적 시각화 + 어제·오늘 outbox에서 뽑은 오늘 한 줄 후보

본인이 매일 새 주제를 찾는 게 아니라 어제의 패턴이 오늘의 시작점으로 자연스럽게 흘러나오는 ritual.

## 0단계 — 오늘 날짜 잡기

`date +%Y-%m-%d`로 오늘 날짜를 잡는다. 요일 감지(`date +%u` — 1=월 〜 7=일)도 같이.

## 1단계 — outbox 전체 스캔

`outbox/*.md` 전체를 읽는다 (변경사항 감지 없이 매번 전체 스캔 — v1 단순성. 100편 규모까지 비용 0).

각 파일에 대해 다음을 추출:

- **slug** — 파일명에서 `.md` 제거
- **claim** — 본문 첫 줄(H1 제외 첫 의미 단락 또는 `> **🧭 첫 줄 주장:**` placeholder)
- **tags** — 본문에서 자연스럽게 뽑은 키워드 2〜4개. frontmatter `tags`가 있으면 그걸 우선
- **메타인지** — 본문 안에서 본인이 반복한 패턴 한 줄. 같은 단어·같은 좌절·같은 결정 패턴을 찾는다. 없으면 "이번에는 새 단어가 나왔다 — {그 단어}." 형식
- **열린 질문** — 본문 안에서 본인이 답을 미뤄둔 질문 한 줄. 없으면 본문이 너무 단정적이라는 신호니까 "정말 그런가?" 류 한 줄

placeholder 파일(첫 줄 주장이 `> **🧭 첫 줄 주장:**` 박스로만 남고 본문이 비어 있는 경우)은 메타인지·열린 질문을 `_(본문 비어 있음)_`로 둔다.

## 2단계 — `.ai-wiki/index.md` 갱신

### 페이지 섹션 갱신

각 outbox 파일에 대해 주 tag의 섹션을 찾아 해당 줄을 덮어쓴다 (없으면 추가). 형식:

```text
### {tag}
- [{slug}](../outbox/{파일명}) — {claim} #{tag1} #{tag2} [type: reflected]
  - 메타인지: {1단계에서 뽑은 한 줄}
  - 열린 질문: {1단계에서 뽑은 한 줄}
```

새 tag가 등장하면 알파벳 순으로 새 섹션 만든다. 같은 path의 기존 줄은 **덮어쓴다** — 인덱스는 현재 상태를 반영한다.

### Log 섹션 append

`## Log` 섹션 맨 위에 한 줄:

```text
- {YYYY-MM-DD HH:MM}: `/일일회고` sync {갱신한 파일 수}편
```

### 인덱스 초기 골격 안내

`index.md`의 `> 아직 비어 있음 — ...` 안내 문구가 있으면 첫 항목 추가 시 같이 제거한다.

## 3단계 — 시각화 데이터 만들기

다음 세 줄만 추출 (v1은 짧게 시작, 알림 피로 방지):

### 누적 글 수

- 전체 `outbox/*.md` 파일 수
- 지난 7일 새 글 수(파일명 날짜 prefix 기준)

### 어제 새 글

- 어제 날짜 prefix의 outbox 파일이 있으면 그 글의 claim 한 줄 + 파일명
- 없으면 "어제는 새 글이 없습니다."

### 오늘 한 줄 후보

어제·오늘 outbox 글에서 가장 자주 부딪힌 한 문장을 뽑는다. 추출 로직:

- 어제·오늘 outbox 글들의 첫 줄 주장(claim)을 본다
- 같은 단어·같은 결정 패턴이 두 번 이상 나오면 그 문장을 후보로 박는다
- 없으면 어제 새 글의 claim 그대로 박는다 (어제의 자연스러운 이어쓰기)
- 어제도 새 글이 없으면 "지난 7일 가장 빈도 높은 키워드"로 한 줄 만들기

### (옵션) 월요일 한정 — 7일 누적

`date +%u`가 `1`(월요일)이면 추가 섹션:

- 지난 7일 outbox에서 가장 많이 등장한 키워드 top 3
- 7일 이상 안 닫힌 열린 질문 (`.ai-wiki/index.md`의 `메타인지`·`열린 질문` 중 같은 줄이 7일 전과 동일하면 "안 닫힘"으로 판정)

월요일이 아니면 이 섹션 생략.

## 4단계 — 출력

### 슬랙 webhook 있는 경우

환경 변수 `SLACK_WEBHOOK_URL`이 설정되어 있으면 슬랙으로 전송. 메시지 형식:

```
📊 outbox 현황 ({YYYY-MM-DD} 기준)
- 누적 글: {N}편 (이번 주 +{M}편)
- 어제 새 글: "{claim}" ({파일명})

✏️ 오늘 한 줄
어제·오늘 outbox에서 가장 자주 부딪힌 한 문장:
> "{후보 한 줄}"

오늘은 이 한 줄 위에서 시작해보세요. `/쓰기`로 글 한 편 열기 →
```

월요일이면 위 메시지에 다음 섹션 추가:

```
📅 지난 7일 패턴
- 자주 쓴 키워드: #{tag1} ({n1}회) · #{tag2} ({n2}회) · #{tag3} ({n3}회)
- 7일 안 닫힌 열린 질문: "{질문 한 줄}" ({처음 등장일} ~ )
```

전송 명령(bash):
```bash
curl -X POST -H 'Content-type: application/json' \
  --data "{\"text\": \"{위 메시지}\"}" \
  "$SLACK_WEBHOOK_URL"
```

### 슬랙 webhook 없는 경우

stdout으로 같은 내용 출력. 또는 `.inbox/일일회고-{YYYY-MM-DD}.md` 파일로 저장 (사용자가 나중에 본인 손으로 다시 본다).

## 5단계 — 사용자에게 한 줄 보고

```
✓ .ai-wiki/index.md 동기화: {갱신 파일 수}편 · Log +1줄
✓ "오늘 한 줄" 가이드 출력: {slack | stdout | .inbox/일일회고-{날짜}.md}

오늘 한 줄: "{후보 한 줄}"
```

## 주의 사항

- `outbox/` 본문은 **읽기만** 한다 — 절대 수정·추가하지 않는다. append-only 가드.
- `.inbox/`는 webhook 없을 때 일일회고 결과 저장용으로만 사용. raw 자료와 섞이지 않게 파일명 prefix `일일회고-`로 구분.
- `.ai-wiki/index.md`의 같은 path 항목은 매 호출 덮어쓴다 — 현재 상태 반영.
- placeholder 상태 outbox(본문 안 채워진 글)는 메타인지·열린 질문을 강제로 짜내지 않는다 — `_(본문 비어 있음)_` 표시.

## 스케줄 등록 — 매일 아침 자동 실행

### macOS launchd 예시

`~/Library/LaunchAgents/com.makerjun.daily-reflect.plist` 작성 (필요 시 사용자가 직접 만든다):

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key><string>com.makerjun.daily-reflect</string>
  <key>ProgramArguments</key>
  <array>
    <string>/bin/bash</string>
    <string>-lc</string>
    <string>cd ~/git/my-brain-v1 && claude --print "/일일회고"</string>
  </array>
  <key>StartCalendarInterval</key>
  <dict>
    <key>Hour</key><integer>7</integer>
    <key>Minute</key><integer>30</integer>
  </dict>
  <key>StandardOutPath</key><string>/tmp/daily-reflect.log</string>
  <key>StandardErrorPath</key><string>/tmp/daily-reflect.err</string>
</dict>
</plist>
```

로드: `launchctl load ~/Library/LaunchAgents/com.makerjun.daily-reflect.plist`

### GitHub Actions 예시 (cross-platform)

`.github/workflows/daily-reflect.yml`:

```yaml
name: Daily Reflect
on:
  schedule:
    - cron: '30 22 * * *'  # 매일 UTC 22:30 = KST 07:30
  workflow_dispatch:
jobs:
  reflect:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run /일일회고 logic
        env:
          SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
        run: bash .claude/scripts/daily-reflect.sh
      - name: Commit index.md changes
        run: |
          git config user.name "github-actions"
          git config user.email "actions@github.com"
          git add .ai-wiki/index.md
          git diff --staged --quiet || git commit -m "chore: /일일회고 sync"
          git push
```

> GitHub Actions 경로를 쓰려면 `.claude/scripts/daily-reflect.sh`에 본 커맨드 로직을 bash로 옮겨 두어야 한다. v1에서는 macOS launchd가 더 단순 — 본인 환경에서 Claude Code를 그대로 호출하므로 옮길 게 없다.

### 슬랙 webhook 발급

1. https://api.slack.com/messaging/webhooks 에서 incoming webhook URL 발급
2. 본인 환경에 `SLACK_WEBHOOK_URL` 환경 변수로 등록 (`~/.zshrc` 또는 launchd plist `EnvironmentVariables`)
3. 본 커맨드는 환경 변수가 있으면 자동으로 슬랙 경로 사용

webhook이 없어도 커맨드는 동작한다 — stdout으로 출력하거나 `.inbox/일일회고-{날짜}.md`로 저장.
