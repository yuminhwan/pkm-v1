# /쓰기

이 커맨드는 새 글 하나를 `outbox/`에 만든다. **빈 페이지 공포 없이 글 한 편을 시작하게 하는 게 목적**이지, AI가 글을 대신 써주는 게 아니다. 첫 줄 주장과 본문은 본인이 채워야 한다 — 그게 generation effect다.

## 0단계 — 두 가지만 묻는다

다음을 **하나씩 순서대로** 묻는다. 한 번에 둘을 묻지 않는다 (인지 부하 X).

1. **"오늘 어떤 의도로 글을 쓰려고 해요? 한 줄로 — 아직 다듬어지지 않아도 OK."**
2. **"파일명에 박을 키워드 한 단어는? (예: `pkm-함정` · `회의-소외감` · `llm-환각`)"**

답을 받으면 다음 단계로.

## 1단계 — 파일 생성

`date +%Y-%m-%d`로 오늘 날짜를 잡고 `outbox/{YYYY-MM-DD}-{키워드}.md`를 만든다. 같은 이름이 이미 있으면 덮어쓰지 않고 사용자에게 알린 뒤 다른 키워드 한 번 더 묻는다.

## 2단계 — 6원칙 골격 박기

본문 형식은 다음과 같이 정확히:

```markdown
# {사용자가 적은 의도를 그대로 제목으로 — 거칠면 거친 대로}

> **🧭 첫 줄 주장 (한 문장으로, 본인 손으로 다듬어 채워주세요):**
> {0단계에서 받은 의도 한 줄을 그대로 placeholder로}

---

## 어떻게 그 결론에 도달했나

- 어떤 사건이 있었나
- 어떤 판단이 바뀌었나
- 어떤 대화가 불편했나

(📖 팩트가 아니라 **서사로**. 시간 순서로 풀어도 OK, 결론부터 거꾸로 풀어도 OK.)

## 다음에 무엇을 할 것인가

- 이 통찰이 내 행동을 어떻게 바꾸는가
- 1주일 안에 시도해볼 한 가지

---

## 6원칙 체크 (글 다 쓰고 본인이 한 번 짚기)

- [ ] 🧭 첫 줄이 가장 강력한 주장으로 정착됐다
- [ ] 📝 한 단락 = 한 아이디어
- [ ] 🔍 파일명이 인출 단서로 떠오른다
- [ ] 📖 팩트 나열이 아니라 서사가 들어 있다
```

## 3단계 — `.ai-wiki/index.md`에 placeholder 한 줄 박기

LLM이 Query 시 가장 먼저 읽는 카탈로그에 이번 새 글의 자리를 미리 잡는다. 본문 미러는 만들지 않는다 — v1에서는 본문 단일 진실원(`outbox/`).

### 페이지 섹션 갱신

사용자 의도 한 줄에서 자연스럽게 뽑은 주 tag로 섹션을 찾는다. 없으면 알파벳 순으로 새 섹션 만든다. 해당 섹션 아래에 한 줄 추가:

```text
### {tag}
- [{파일명에서 슬러그 추출}](../outbox/{파일명}) — {사용자 의도 한 줄 그대로} #{tag1} [type: reflected]
  - 메타인지: _(pending — /일일회고가 본문 보고 채움)_
  - 열린 질문: _(pending — /일일회고가 본문 보고 채움)_
```

claim 자리에 일단 사용자 의도 한 줄을 그대로 박는다. 사용자가 본문을 다 쓰고 첫 줄 주장을 다듬으면 다음 `/일일회고`가 본문 첫 줄을 읽어 claim을 최신화한다.

### Log 섹션 append

`## Log` 섹션 맨 위에 한 줄:

```text
- {YYYY-MM-DD HH:MM}: `/쓰기` new {파일명} ("{사용자 의도 한 줄}")
```

### 인덱스 초기 골격 안내

`index.md`의 `> 아직 비어 있음 — ...` 안내 문구가 있으면 첫 항목 추가 시 같이 제거한다.

## 4단계 — 한 줄 보고

```
✓ 새 글 골격: outbox/{YYYY-MM-DD}-{키워드}.md
✓ .ai-wiki/index.md 갱신: +1 placeholder · Log +1줄

첫 줄 placeholder에 들어간 의도: "{사용자 의도 한 줄 그대로}"

이제 본인 손으로 첫 줄 주장을 다듬고 본문을 채우면 됩니다.
한 단락도 AI에 통째로 던지지 마세요 — generation effect는 그 단위에서 일어납니다.
내일 아침 /일일회고가 본문을 읽고 메타인지·열린 질문을 채워줍니다.
```

## 주의 사항

- **AI가 본문을 채우지 않는다.** 첫 줄 placeholder는 사용자 의도 그대로 옮기는 정도, 다듬기·재작성 X.
- **묻는 건 두 가지뿐.** 더 묻지 않는다. 빈 페이지보다 *시작점이 잘못된 페이지*가 더 쉽다 — 일단 골격만 박고 사용자가 채우게 한다.
- 같은 키워드 파일이 이미 있으면 덮어쓰지 않고 다른 키워드를 받는다.
- `.inbox/`는 절대 건드리지 않는다.
- `.ai-wiki/`는 `index.md`에 한 줄만 추가, 본문 미러 파일은 만들지 않는다.
- 추가 안내·요약·정리를 본인이 알아서 더 붙이지 않는다. 골격 + 인덱스 한 줄 + 한 줄 보고로 끝낸다.
