# /주간회고

주간 회고를 진행하고, **회고에서 나온 것을 실제 행동으로 옮겼는지** 코칭합니다.

## 목표

사용자의 핵심 주장: **"협업을 LLM을 통해서 어떻게 하면 잘 할 수 있을 지 생각해보고 적용해본다"**

이 커맨드는 회고를 평가하는 게 아니라, **회고에서 나온 다짐을 실제 행동으로 옮기는 루프**를 돕는다.

## 1단계 — 회고 질문

사용자에게 네 가지를 **하나씩 순서대로** 묻는다.

1. **이번 주 잘한 것** — 협업, 소통, 작업 중 잘했다고 느끼는 것
2. **이번 주 아쉬운 것** — 개선할 수 있었다고 느끼는 것 (구체적으로)
3. **지난주 다짐 적용 여부** — 지난주에 "다음 주 시도할 것"으로 정한 걸 실제로 했는가?
4. **다음 주 시도할 것** — 실제로 해볼 수 있는 작은 변화

각 질문에 대해 사용자가 답변하면 다음 질문으로 넘어간다.

## 2단계 — 파일 저장

`date +%Y-%m-%d`로 오늘 날짜를 잡고 `outbox/{YYYY-MM-DD}-주간회고.md`를 만든다.

```markdown
# 주간 회고 — {YYYY-MM-DD}

## 이번 주 잘한 것
{사용자 답변}

## 이번 주 아쉬운 것
{사용자 답변}

## 지난주 다짐 적용 여부
{사용자 답변}

## 다음 주 시도할 것
{사용자 답변}

---

## LLM 코칭
{서브에이전트가 코칭한 내용}
```

## 3단계 — 코칭 서브에이전트 실행

아래 루브릭 기반으로 서브에이전트를 실행하여 코칭 결과를 파일에 추가한다.

### 평가 루브릭 (행동 전환 관점)

| 항목 | 질문 | 1~5점 |
|------|------|-------|
| **구체성** | 아쉬운 점이 구체적인가? (예: "소통 부족" → "주간 회의 때 질문 못함") | 1: 모호함 → 5: 매우 구체적 |
| **다짐** | 다음 주 시도할 것이 실행 가능한가? (작고 명확한 행동인가) | 1: 막연함 → 5: 즉시 실행 가능 |
| **적용** | 지난주 다짐을 실제로 적용했는가? | 1: 안 함 → 5: 완료 |
| **피드백** | 팀원에게 피드백을 구했는가? (직접 물어봄, 관찰 등) | 1: 안 함 → 5: 적극적으로 구함 |

### 코칭 결과 형식

```markdown
### LLM 코칭

**이번 주 점수**: {X}/20점

**항목별 점수**:
- 구체성: {X}/5 — {한 줄 코멘트}
- 다짐: {X}/5 — {한 줄 코멘트}
- 적용: {X}/5 — {한 줄 코멘트}
- 피드백: {X}/5 — {한 줄 코멘트}

**이번 주 한 줄**: {이번 주 상태 요약}

**다음 주 코칭 제안**: {구체적 실행 제안 1~2가지}
```

## 4단계 — 결과 보고

```
✓ 주간 회고 완료: outbox/{YYYY-MM-DD}-주간회고.md
✓ 점수: {X}/20점

{코칭 결과 요약}
```

```

사용자의 회고:

다음 루브릭으로 평가하고 코칭 결과를 위 형식으로 작성하세요:

| 항목 | 질문 | 점수 |
|------|------|------|
| 구체성 | 아쉬운 점이 구체적인가? | 1~5점 |
| 다짐 | 다음 주 시도할 것이 실행 가능한가? | 1~5점 |

```
