# 코틀린의 예외 처리를 자바와 비교하면서 하나씩 알아보고 싶다

> **🧭 첫 줄 주장 (한 문장으로, 본인 손으로 다듬어 채워주세요):**
> 코틀린의 예외 처리를 자바와 비교하면서 하나씩 알아보고 싶다


현재 프로젝트를 코틀린 기반으로 진행하고 있다. 자바에 대해서는 문법이나 원리에 대해서 익숙하지만 코틀린에 대해서는 제대로 파악을 하지 못하고 있는 것 같다. 
AI를 통해 개발을 한다고 했지만 작성한 코드가 알맞은 코드인가를 판단하기엔 지식이 부족하다고 느꼈다. 

AI가 개발을 잘 해주지만 그 개발을 잘하기 위한 기준은 내가 알고 있어야 판단을 내릴 수 있을 것 같고 이를 통해 메모리나 훅 등으로 방지할 수 있을 듯 하다. 

공식문서를 기준으로 자바와 비교해서 어떤 차이점이 있는 지 파악해보고 학습해보고자 한다. 

## 카프카 예외 처리 

자바와 동일하게 try - catch문을 사용할 수 있다. 

다만, 코틀린에는 Result, runCatching을 통해 예외 처리가 가능하다. 
Result는 성공 / 실패를 나타내고 runCatching은 Result를 반환하게 된다.
```kotlin 
@JvmInline value class Result<out T> /* internal constructor */ {
    val isSuccess: Boolean
    val isFailure: Boolean
    fun getOrNull(): T?
    fun exceptionOrNull(): Throwable?
    
    companion object {
        fun <T> success(value: T): Result<T>
        fun <T> failure(exception: Throwable): Result<T>
    }
}

@InlineOnly
@SinceKotlin("1.3")
public inline fun <R> runCatching(block: () -> R): Result<R> {
    return try {
        Result.success(block())
    } catch(e: Throwable) {
        Result.failure(e)
    }
}
```

- onSuccess : 성공 
- onFailure : 실패 
- recover :  새로운 값을 넣어서 Result를 복구
- getOrNull : 에러 무시하고 null 반환
- getOrDefault : 에러 무시하고 기본값 반환 
- getOrElse
- exceptionOrNull

Result를 그대로 반환함으로써 각 사용처에서 원하는 대로 처리할 수 있다.
- A 서비스에서는 기본값으로 대체 
- B 서비스에서는 예외 던짐
