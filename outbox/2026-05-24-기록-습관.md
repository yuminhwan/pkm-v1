# 지식관리를 위한 기록을 잘하고 싶은데 어떻게 해야할지 잘 모르겠다

> **🧭 첫 줄 주장 (한 문장으로, 본인 손으로 다듬어 채워주세요):**
> 지식관리를 위한 기록을 잘하고 싶은데 어떻게 해야할지 잘 모르겠다.
 
업무를 하거나 학습을 한 것에 대해서 기록 및 정리를 하지 않다보니 이력서를 업데이트할 때 많은 어려움이 있었다. 업데이트를 위해 한 일에 대해서 생각해보아도 제대로 생각나지 않는 것들이 많았고 내가 작성한 이슈나 PR 를 봐야지 조금 기억만 날 정도이고 당시에 했던 의사 결정, 고민, 인사이트는 생각이 나지 않아 이력서 작성에 어려움을 겪었다. 

지식 관리에 대해 설명을 들었을 때 일단 기록이 제일 최우선 이기 때문에 어떤 형태로든 일단 기록하는 것이 중요하다고 생각이 들었다. 돌이켜보면 잘 구조화해서 적어야 한다는 명목에 기록을 하지 않고 나중에 해야지 라고 생각했던 점이 많았던 거 같다. 요즘에는 LLM으로 구조화도 잘 시켜주니 일단 키워드나 raw 하더라도 그때의 생각을 잘 들어날 수 있도록 기록을 해두는 것이 중요하다고 생각했다. 이후에 해당 글을 좀 더 구조화시켜주도록 LLM와 핑퐁하면서 만들 수 도 있을 것 같다. 

왜 그랬을까? 
정리할 때 한번에 제대로 알고 정리해야한다는 강박?이 있었던 것 같다. 그러다보니 기록을 시작부터 못했던 것 같다.  
이를 해결하기 위해서는 일단 학습을 하거나 일을 했을 때 그거에 대한 내용을 그냥 마구잡이로 일단 기록을 하는 것이 좋을 것 같다. 참고했던 글이나 링크, 사진 등도 막 넣어두고 이를 정리하는 것은 나중에 생각해봐야 겠고 하루 단위로 내가 뭘 했고 다음날은 뭘 해야하고 이를 통해서 배운 점이나 기록할 만 점을 짧게 라도 정리하는 시간을 가져봐야겠다.
