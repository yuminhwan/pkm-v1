# pkm-v1

> 안드레 카파시의 "LLM wiki" 패턴을 따라 만든 한국어 개인 지식관리(PKM) 템플릿. 작지만 유용한 v1.

Fork해서 본인의 brain으로 굴리는 starter 저장소. 폴더 세 개 · 슬래시 커맨드 세 개 · 인덱스 한 장 — 그게 전부다. 원본은 [`imakerjun/my-brain-v1`](https://github.com/imakerjun/my-brain-v1)에서 시작되었다.

## 한 줄 메커니즘

**사람은 `outbox/`에만 쓴다. LLM은 `.ai-wiki/index.md`를 먼저 읽어 어느 본문을 펼칠지 정한다. 둘 사이는 `/쓰기`(새 글)·`/수집`(raw)·`/일일회고`(매일 아침 자동 회고)가 채운다.**

> **LLM 진입 안내**: 새 세션은 가장 먼저 [`.ai-wiki/index.md`](.ai-wiki/index.md)를 읽어 카탈로그를 잡는다. 본문은 `outbox/`·`.inbox/`에 단일 진실원으로 있고, 인덱스 한 장이 어느 페이지를 펼칠지 결정한다. 임베딩 벡터 검색 없이 마크다운 한 장으로 RAG를 대체하는 카파시 LLM wiki 패턴.

## 폴더 의미

| 폴더 | 누가 쓰나 | 무엇이 들어가나 |
|---|---|---|
| `outbox/` | **사람** | 사람 머리로 소화해서 만든 출력. 글·노트·정리. 본인 손맛이 남아 있는 글. 매일 손이 가는 1순위 자리. |
| `.inbox/` | 사람 (raw 던지기) | raw 자료를 던져두는 임시 자리. 스크린샷·링크·메모·녹취록·AI 답변까지 전부. 가공 X. `/수집 <URL\|텍스트>`로 한 줄에 던질 수도 있다. |
| `.ai-wiki/` | 커맨드 (자동) | LLM이 가장 먼저 읽는 카탈로그(`index.md`)만 둔다. 본문 미러 X — `outbox/`·`.inbox/`의 본문을 가리키는 한 줄씩만 박힘. `type: raw`와 `type: reflected` 두 종류가 공존하고, reflected만 메타인지·열린 질문 두 줄을 추가로 가진다. 손으로 안 만진다. |

> 안드레 카파시의 표현을 빌리면: 가장 강력한 노트는 future me · current me · LLM 셋 다를 동시에 만족시키는 노트다. `outbox/`는 사람 둘을 위해, `.ai-wiki/index.md`는 LLM을 위해 같은 글에 두 번째 시각(인덱스)을 박는다.

## 왜 점(`.`) prefix인가

점으로 시작하는 폴더(`.claude`, `.inbox`, `.ai-wiki`)는 **본인이 직접 안 건드리고 도구·커맨드가 관리하는 자동화 영역**이라는 의미가 Unix 컨벤션으로 박혀 있다. prefix 없는 `outbox/`만 사람이 매일 손으로 쓰는 자리.

```bash
ls          # → outbox/ 한 곳만 보임 (사람용)
ls -a       # → .inbox/ .ai-wiki/ .claude/ 까지 모두 보임 (자동화 포함)
```

`ls` 기본에서 `outbox/` 한 곳만 보이는 게 **의도된 우선순위 표현**. 1순위가 자연스럽게 눈에 들어오고 보조 영역은 숨어 있다.

## 슬래시 커맨드

`.claude/commands/` 안에 세 개 들어 있다.

| 커맨드 | 무엇을 하나 |
|---|---|
| `/쓰기` | 새 글 한 편을 `outbox/`에 만든다. 두 가지(오늘의 의도 한 줄 · 파일명 키워드)만 묻고 6원칙 골격을 박는다. 본문은 본인이 채운다. 완료 시 `.ai-wiki/index.md`에 placeholder 한 줄 박힘 |
| `/수집` | URL·텍스트를 `.inbox/`에 raw로 저장 + `.ai-wiki/index.md`에 한 줄 등록. 다음 글 쓸 때 LLM이 이 자료를 카탈로그에서 바로 잡는다 |
| `/일일회고` | 매일 아침 자동(스케줄)으로 실행. outbox 전체를 스캔해 `.ai-wiki/index.md`의 claim·메타인지·열린 질문을 최신화 + 누적 시각화·오늘 한 줄 가이드를 슬랙(또는 stdout)으로 보낸다. 수동 호출도 가능 |

두 빈 공포가 각각의 커맨드와 짝을 이룬다 — `/쓰기`는 빈 페이지, `/수집`은 빈 인박스. `/일일회고`가 매일 아침 보내주는 "오늘 한 줄" 가이드가 다음 글의 시작점이 된다. **만드는 행위 = 어제의 나, 받는 행위 = 오늘의 나** — 같은 글이 시간 차로 두 번 만난다.

## 시작하기 — 첫 outbox 만들기

1. Claude Code에서 `/쓰기` 입력
2. 두 질문에 답한다 — *오늘 어떤 의도로 글을 쓰려고 해?* · *파일명 키워드 한 단어는?*
3. 생긴 파일을 열어 6원칙 골격 위에 본문을 본인 손으로 채운다 (AI에 단락 통째 안 던지기)
4. `.ai-wiki/index.md`를 열어 본인 글이 페이지 섹션에 한 줄 박힌 걸 확인
5. 다음 날 아침 `/일일회고` 실행 — outbox 변경사항이 카탈로그에 반영되고 "오늘 한 줄" 가이드를 받는다

> `outbox/첫-outbox.md`는 *한 글이 어떤 형태로 정착하는지* 보여주는 reference 템플릿. `/쓰기`가 만드는 골격과 같은 형식이지만 본인이 한 번 직접 손으로 본 다음에 `/쓰기`로 매일 굴리는 게 자연스러운 흐름.

## outbox 6원칙

`outbox/`에 글을 쓸 때만 신경 쓰면 되는 형식. `.ai-wiki/`는 동기화가 자동으로 챙긴다.

| 원칙 | 왜 이렇게 쓰나 |
|---|---|
| 🧭 첫 줄이 가장 강력한 주장 | 한 문장으로 못 쓰면 그 개념을 아직 모르는 것 — 자기 점검 (Feynman 방식) |
| 📝 한 단락 한 아이디어 | 단일 아이디어 노트 (Zettelkasten의 atomic note 원칙) |
| 🔗 위키·상대경로 링크 (순환 OK) | 노트끼리 연결되면서 통찰이 자연스레 자람 |
| 🔍 파일명 = 인출 단서 | 파일명이 안 떠오르면 이름을 잘못 지은 것 |
| 📂 평면 배치 (서브폴더 거의 안 씀) | 같은 폴더 안에서 다른 노트가 자동으로 보이면서 우연한 발견이 일어남 |
| 📖 사실이 아니라 이야기 | 서사 단위로 저장되는 기억 — 이야기로 풀어야 한 달 뒤에도 살아남음 |

`.ai-wiki/`는 손으로 만지지 않는다. 만지고 싶은 문장이 생기면 `outbox/`로 가서 새 글로 쓴다.
