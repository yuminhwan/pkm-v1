# [Kotlin] 코틀린을 입문할 때 알면 좋은 내용들 모음

## 코틀린 공통

### 코틀린의 설계 목적

코틀린은 간결성이 아닌 **가독성(readability)**을 높이는 데 목표를 두고 설계된 프로그래밍 언어입니다. 코드를 읽는 것이 쓰는 것보다 중요하므로, 항상 인지 부하를 줄이는 방향으로 코드를 작성해야 합니다.

### 코틀린의 컴파일러

코틀린 컴파일러는 JVM에서 실행될 수 있는 바이트코드를 생성합니다. 자바 코드를 생성하는 것이 아니기 때문에, 자바와 다르게 체크 예외를 검사하지 않으며 모든 예외를 언체크 예외로 인식합니다.

### 불변성(Immutability)

코틀린은 변경 가능성보다 불변성을 선호합니다. 클래스는 기본적으로 final이며, 변수도 기본적으로 불변입니다. 초기화 후 변경되지 않으면 `val`을 사용하고, 가능하면 불변 컬렉션 인터페이스를 사용하세요.

### null 처리: 안전 호출과 엘비스 연산자

코틀린은 기본적으로 null을 허용하지 않습니다. nullable 타입은 타입 뒤에 `?`를 붙여 표현합니다.

```kotlin
fun length(): Int? = null

// 안전 호출과 엘비스 연산자
var p = Person(first = "North", middle = null, last = "West")
val middleNameLength = p.middle?.length ?: 0
```

연쇄 호출을 묶을 때는 다음 줄에 `.` 또는 `?.`를 한 줄 들여쓰기로 넣는 것을 권장합니다.

```kotlin
val anchor = owner
    ?.firstChild!!
    .siblings(forward = true)
    .dropWhile { it is PsiComment || it is PsiWhiteSpace }
```

### 원시 타입과 래퍼 타입

코틀린은 원시 타입과 래퍼 타입을 구분하지 않습니다. 대부분의 경우 `Int` 타입은 자바의 `int` 원시 타입으로 컴파일되지만, null을 반환하거나 제네릭 클래스를 사용하면 래퍼 타입이 사용됩니다.

### if와 when 문법

이진 조건에서는 `if`를, 3개 이상의 선택지에서는 `when`의 사용을 권장합니다.

```kotlin
if (x == null) ... else ...

when (x) {
    ACTIVE -> // ...
    INACTIVE -> // ...
    ELSE -> // ...
}
```

### 플랫폼 타입(Platform Type)

자바에서 호출된 타입이 nullable 한지 알 수 없을 때, `String!` 같이 `!` 기호를 붙여 표기합니다. 직접 조작 가능한 자바 코드라면 `@Nullable`과 `@Notnull`을 붙여주는 것이 좋습니다.

### 네임드 아규먼트(Named arguments)

동일한 타입의 매개변수가 많거나 단위를 명확히 알 수 없는 경우 네임드 아규먼트를 사용하세요.

```kotlin
sleep(timeMillis = 100)
drawSquare(x = 10, y = 10, width = 100, height = 100, fill = true)
```

### object 키워드

싱글턴을 간단하게 정의할 수 있습니다.

```kotlin
object StringUtils {
    fun toLowerCase(input: String): Boolean {
        return input.toLowerCase()
    }
}
```

클래스 내부의 `companion object`는 상수와 팩터리 메서드를 담을 때 사용합니다.

```kotlin
@Service
class SignUpService {
    fun signUp(input: SignUpInput) {
        delaySignUp(DELAY_MINUTES)
    }

    companion object {
        private const val DELAY_MINUTES: Long = 5
    }
}
```

### by 대리자

다른 클래스의 인스턴스에 연산을 위임할 때 사용합니다.

```kotlin
data class Cards(
    val cards: List<Card>,
) : List<Card> by cards {
    fun hasActiveCard(user: User): Boolean {
        return cards.any{ it.status == CardStatus.ACTIVE }
    }
}

val isEmpty = cards.isEmpty()
```

### lazy 대리자

속성의 초기화를 지연시킬 때 사용합니다.

```kotlin
data class Student(
    val name: String,
    var score: Int,
) {
    val emptySize: Int by lazy(LazyThreadSafetyMode.SYNCHRONIZED) {
        println("@@@@@@Initialize by lazy")
        score
    }
}
```

### 연산자 오버로딩

산술 연산자, 단항 연산자, get/set, in, 비교 연산자 등을 오버로딩할 수 있습니다.

```kotlin
data class Point(val x: Int, val y: Int) {
    operator fun plus(other: Point): Point {
        return Point(x + other.x, y + other.y)
    }
}
```

Infix 키워드를 통한 중위 연산자도 제공됩니다.

```kotlin
infix fun Int.plus(n: Int): Int {
    return this + n
}

fun main() {
    println(5 plus 10)
}
```

## 함수

### 함수의 선언

백틱을 활용한 메서드 선언은 테스트에서 허용됩니다.

```kotlin
class MyTestCase {
    @Test fun `ensure everything works`() { /*...*/ }
    @Test fun ensureEverythingWorks_onAndroid() { /*...*/ }
}
```

메서드 이름은 다음을 따르세요:
- 객체 변경 여부를 이름에 명시 (`sort` vs `sorted`)
- 무의미한 이름(Manager, Wrapper) 지양
- 약어는 두 글자면 대문자, 더 길면 첫 글자만 대문자 (IOStream, HttpInputStream)

### Expression body 표현식

간단한 함수라면 expression body 사용을 권장합니다.

```kotlin
fun foo(): Int {     // bad
    return 1
}

fun foo() = 1        // good

fun f(x: String, y: String, z: String) =
    veryLongFunctionCallWithManyWords(andLongParametersToo(), x, y, z)
```

### 디폴트 파라미터(Default Parameter)

불필요한 메서드 오버로딩을 줄일 수 있습니다.

```kotlin
// Bad
fun foo() = foo("a")
fun foo(a: String) { /*...*/ }

// Good
fun foo(a: String = "a") { /*...*/ }
```

자바에서 코틀린 함수를 호출하는 경우, `@JvmOverloads` 애노테이션을 추가하면 자동으로 오버로딩 메서드가 생성됩니다.

### 함수형 메서드

자주 사용되는 함수형 메서드들:
- `withIndex`: 인덱스와 값을 함께 순회
- `associate`, `associateWith`: 맵 생성
- `ifEmpty`: 빈 컬렉션에 기본값 처리
- `sortedWith & compareBy`: 다중 속성 정렬
- `chunked`: 같은 크기로 분할
- `windowed`: 정해진 간격으로 이동
- `filterIsInstance`: 타입으로 필터링

람다 파라미터는 짧고 중첩되지 않으면 `it` 키워드로 접근하세요.

```kotlin
val result = listOf("A", "B").filter { it.startsWith("A") }
```

### 시퀀스(Sequence)의 사용

Iterable은 처리 연산마다 새 컬렉션을 생성하지만, Sequence는 지연(lazy) 처리를 지원합니다.

```kotlin
val result = listOf("A", "B")
    .asSequence()
    .filter { it.startsWith("A") }
    .filter { it.startsWith("B") }
```

시퀀스의 장점:
- 자연스러운 처리 순서 유지
- 최소한만 연산
- 무한 시퀀스 형태 사용 가능
- 각 단계에서 컬렉션 생성 없음

### 자바 스트림과 코틀린 시퀀스의 차이

- 코틀린 시퀀스가 더 많은 처리 함수 제공
- 자바 스트림은 병렬 모드 지원
- 코틀린 시퀀스는 모든 플랫폼에서 사용 가능

일반적인 경우 코틀린 시퀀스를, 병렬 처리가 필요한 경우만 자바 스트림을 사용하세요.

### 영역 함수(scope function)

특정 객체의 컨텍스트 내에서 코드 블록을 실행합니다.

```kotlin
Person("Alice", 20, "Amsterdam").let {
    println(it)
    it.moveTo("London")
    it.incrementAge()
    println(it)
}
```

영역 함수들의 선택 기준:

![scope functions comparison table]

**let**
- 호출 chain의 결과를 바탕으로 연산 필요
- null이 아닌 값에 대한 연산 필요
- 제한된 범위의 로컬 변수 도입

```kotlin
val numbers = mutableListOf("one", "two", "three", "four", "five")
numbers.map { it.length }.filter { it > 3 }.let {
    println(it)
}

val str: String? = "Hello"
val length = str?.let {
    println("let() called on $it")
    processNonNullString(it)
    it.length
}
```

**with**
- "이 객체를 사용하여 다음을 수행합니다"고 읽기 쉽게
- 도우미 객체 도입

```kotlin
val numbers = mutableListOf("one", "two", "three")
with(numbers) {
    println("'with' is called with argument $this")
    println("It contains $size elements")
}

val firstAndLast = with(numbers) {
    "The first element is ${first()}," +
    " the last element is ${last()}"
}
println(firstAndLast)
```

**run**
- 람다 내부에서 객체 초기화 및 반환값 계산
- 식이 요구되는 여러 문이 필요할 때

```kotlin
// 확장 함수
val service = MultiportService("<https://example.kotlinlang.org>", 80)
val result = service.run {
    port = 8080
    query(prepareRequest() + " to port $port")
}

// 일반 함수
val hexNumberRegex = run {
    val digits = "0-9"
    val hexDigits = "A-Fa-f"
    val sign = "+-"
    Regex("[$sign]?[$digits$hexDigits]+")
}
```

**apply**
- 값을 반환하지 않고 멤버에 작업
- 객체 구성에 사용
- 복잡한 처리를 위해 여러 호출 체인 필요

```kotlin
val adam = Person("Adam").apply {
    age = 32
    city = "London"
}
println(adam)
```

**also**
- 객체에 대한 참조 필요
- "추가적인 작업을 수행합니다"

```kotlin
val numbers = mutableListOf("one", "two", "three")
numbers
    .also { println("The list elements before adding new one: $it") }
    .add("four")
```

**takeIf와 takeUnless**
- `takeIf`: 조건 만족 시 값 반환, 아니면 null
- `takeUnless`: 조건 만족 시 null, 아니면 값 반환

### 최상위 함수 선언

코틀린 파일에는 클래스 없이 함수를 선언할 수 있습니다.

```kotlin
var hehe = 10

fun getValue(): Any {
    return 7
}
```

### 확장 함수 (Extension Function)

클래스 밖에 선언되지만 멤버 함수처럼 호출합니다.

```kotlin
fun String.getLengthMinusOne(): Int = this.length - 1

val myLength = "MangKyu".getLengthMinusOne()
```

특정 유스케이스에 종속적인 기능은 해당 유스케이스 내부에 구현하세요.

```kotlin
data class User(
    val userNo: Long,
    val age: Int,
) {
    fun canIssueCard(user: User): Boolean {
        return user.age >= 20
    }
}

@Service
class IssueKbCardService{
    fun canIssueCard(user: User) {
        return user.canIssueKbCard()
    }
}

private fun User.canIssueKbCard(): Boolean {
    return this.age >= 21
}
```

### inline 함수와 reified 키워드

`inline` 키워드를 붙이면 함수 호출이 함수 본문의 바이트코드로 교체됩니다. `reified` 키워드는 inline 함수에서 타입을 보존하기 위해 사용됩니다.

### 예외 처리

코틀린은 `try-catch` 기반의 예외 처리를 지원하며, `use` 키워드로 자바의 try-with-resources를 대체할 수 있습니다.

```kotlin
FileInputStream("file.txt").use {
    print(it)
}
```

`runCatching`으로 더 가독성 좋은 코드를 작성할 수 있습니다.

```kotlin
private fun getResult(id: Long): String? {
    return runCatching {
        call(id)
    }.onFailure {
        log.info("데이터 조회에 실패했습니다.", it)
    }.getOrNull()
}
```

## 클래스

### 파일의 구성

단일 클래스라면 파일명을 클래스 이름으로 하되, 여러 클래스를 포함하면 내용을 설명하는 적절한 이름을 사용하세요.

```kotlin
class Member(
    val id: Long,
    val name: String,
    val status: MemberStatus,
)

enum class MemberStatus {
    ACTIVE,
    INACTIVE,
    LEAVE,
}
```

### 클래스의 가시성

- `private`: package-private으로 컴파일
- `internal`: 같은 모듈 내에서만 호출 가능

### 클래스의 생성자와 초기화 블록

```kotlin
// Primary constructor
class Person(val name: String, var age: Int) {
    // Class body
}

// Secondary constructor
class Person {
    var name: String
    var age: Int

    constructor(name: String, age: Int) {
        this.name = name
        this.age = age
    }

    init {
        require(name.length > 5)
    }
}
```

### 프로퍼티 선언

프로퍼티는 접근자(getter, setter)를 나타내며, `val`은 getter, `var`는 getter/setter를 생성합니다.

```kotlin
class Person(val name: String, var age: Int) {
    val length = name.length

    var priority = 3
    set(value) {
        field = value.coerceIn(1..5)
    }

    val isLowPriority get() = priority < 10
}
```

프로퍼티는 다음 조건에서만 사용하세요:
- throw가 발생하지 않음
- 연산 비용이 저렴하거나 캐싱됨
- 객체 상태가 바뀌지 않으면 같은 결과 반환

### 클래스의 레이아웃

1. 프로퍼티 선언과 초기화 블록
2. 부 생성자
3. 메서드 선언
4. 동반 객체

메서드는 알파벳 순이나 가시성으로 정렬하지 않고, 독자가 따라갈 수 있도록 배치하세요.

### 데이터 클래스

equals, hashCode, toString이 완벽하게 갖춰진 클래스입니다.

```kotlin
data class Member(
    val id: Long,
    val name: String,
)
```

모든 프로퍼티를 읽기 전용으로 만들어 불변으로 설계하고, `copy` 메서드로 일부 속성이 변경된 새 객체를 만드세요.

### Sealed 클래스

다른 클래스의 상속을 제한하는 클래스입니다.

```kotlin
sealed class Animal {
    abstract fun getInfo(): String
}

data class Dog(val name: String) : Animal() {
    override fun getInfo() = "This is a dog named $name."
}

data class Cat(val name: String, val age: Int) : Animal() {
    override fun getInfo() = "This is a cat named $name, and it is $age years old."
}

data class Fish(val color: String) : Animal() {
    override fun getInfo() = "This is a $color fish."
}
```

### 인라인 클래스와 타입 별칭

인라인 클래스는 기본 생성자 프로퍼티가 하나일 때 사용하며, 컴파일 시 프로퍼티로 교체됩니다.

```kotlin
inline class Name(private val value: String) {
}

val name: Name = Name("MangKyu")
// 컴파일 후: val name: String = "MangKyu"
```

타입 별칭으로 길고 반복적인 타입에 새로운 이름을 붙여줄 수 있습니다.

```kotlin
typealias MouseClickHandler = (Any, MouseEvent) -> Unit
typealias PersonIndex = Map<String, Person>
```

## 스프링 프레임워크와의 결합

### kotlin-spring 플러그인

스프링은 프록시 기반 기술을 사용하므로 상속이 필요합니다. 코틀린은 기본적으로 final 클래스이므로, `kotlin-spring` 플러그인으로 스프링 관리 빈을 open 클래스로 만들어주세요.

### data 클래스로 영속성 엔티티 구현

`no-arg` 플러그인으로 기본 생성자를 추가하고, `kotlin-jpa` 플러그인으로 자동 표시합니다. 하지만 양방향 연관 관계에서는 toString, hashCode 호출 시 무한 순환 참조가 발생할 수 있으므로 주의하세요.
