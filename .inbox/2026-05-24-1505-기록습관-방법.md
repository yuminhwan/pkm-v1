## 1. 노트 자체의 구조 — Zettelkasten (Niklas Luhmann)

### (a) Luhmann의 실적

- 독일 사회학자 Niklas Luhmann은 자신만의 노트 시스템 하나로 **50권 이상의 책 + 600편 이상의 논문**을 출판했다.
- 그가 사용한 도구는 종이 카드 박스(Zettelkasten) — 도구가 아니라 *구조*가 핵심이었다.

### (b) 두 가지 핵심 원칙

1. **Atomicity (원자성)** — 카드 한 장에 *하나의 아이디어*만. 여러 개를 묶지 않는다.
2. **Linking (연결성)** — 카드들은 폴더 계층이 아니라 *서로를 가리키는 링크*로 연결된다. 위계가 아니라 그래프.

### (c) 효과

- 일부 분석에서: 폴더 계층 시스템 대비 **연결 노트가 정보 인출률 약 40% 향상**.
- 가장 큰 가치: *예상치 못한 연결*에서 통찰이 발생한다. Luhmann 본인이 자신의 시스템을 "두 번째 기억"이자 "대화 상대"로 묘사.

→ 출처: [Zettelkasten.de — Introduction](https://zettelkasten.de/introduction/), [Zettelkasten.de — Principle of Atomicity](https://zettelkasten.de/posts/principle-of-atomicity-difference-between-principle-and-implementation/), [Todoist Productivity Methods — Zettelkasten](https://www.todoist.com/productivity-methods/zettelkasten-method)

## 2. 흐름 — CODE / PARA (Tiago Forte, *Building a Second Brain*)

### (a) CODE — 노트 생애주기

| 단계 | 활동 |
|---|---|
| **C**apture | 주목을 끄는 모든 것을 일단 담는다 (생각·인용·기사·이미지). 거르지 않는다. |
| **O**rganize | 자료를 PARA로 분류. |
| **D**istill | 핵심을 추출. 패턴·통찰을 발견. *progressive summarization* 사용. |
| **E**xpress | **표현**이 모든 단계의 존재 이유**. 다른 모든 단계는 이걸 위해 존재한다. |

### (b) PARA — 분류 체계

- **P**rojects: 끝이 있는 단기 목표.
- **A**reas: 진행 중인 책임 영역 (건강·커리어·재무 등).
- **R**esources: P·A를 보조하는 자료.
- **A**rchives: 완료·비활성 항목.

### (c) 핵심 강조

- "표현되지 않는 지식은 지식이 아니다" — Forte는 outbox(express)가 없는 PKM은 결국 *디지털 호더링*이라고 진단.
- 직장에서의 적용: 프로젝트별 노트와 reference 자료를 PARA로 분리하면 context switching 비용이 줄어든다.

> 주의: CODE/PARA에 대한 *공식 학술 메타분석*은 없다. Forte 본인과 실무자 사례 기반 — Zettelkasten·인지과학 연구와 달리 empirical evidence가 약한 영역.

→ 출처: [Forte Labs — Building a Second Brain Overview](https://fortelabs.com/blog/basboverview/), [Forte Labs — PARA Method](https://fortelabs.com/blog/para/)

## 3. 습관 자체의 형성 — BJ Fogg, *Tiny Habits* (Stanford)

### (a) Fogg Behavior Model

`B = MAP` — Behavior = Motivation × Ability × Prompt.

- 행동이 일어나려면 동기·능력·프롬프트가 *같은 순간에* 모두 임계치 이상이어야 한다.
- **동기는 파도처럼 변한다** — 동기 높을 때 의지해 큰 행동을 잡으면 다음 주에 무너진다.
- 해결: 동기 낮을 때도 작동하는 *Ability 높은(쉬운) 행동*으로 설계.

### (b) Tiny Habits Recipe

```
After I [기존 행동], I will [새 행동, 30초 이하].
```

이걸 **anchor moment**라고 부른다. Gollwitzer의 implementation intention 연구의 BJ Fogg 변형.

- 예: "양치 후 → 노트 한 줄만 적기"
- 예: "회의 끝나면 → 결정 한 줄만 적기"

### (c) 자동화까지 걸리는 시간

- 충분히 작으면 **2–4주 일일 반복**으로 자동화 가능.
- 일반 습관 형성 범위는 18–254일(Lally et al. 2010) — Tiny Habits가 빠른 이유는 *작아서 매일 성공*하기 때문.

### (d) Celebration (축하) — 자주 빠지는 단계

- 행동 직후 작은 정서적 보상(웃기·"좋아"라고 말하기 등)을 즉시 부여.
- 정서적 강화가 신경 회로를 굳히는 데 결정적 — 시간(반복)보다 *순간의 강도*가 효과적이라는 게 Fogg의 핵심 발견.

→ 출처: [Stanford GSB — Building Habits: The Key to Lasting Behavior Change](https://www.gsb.stanford.edu/insights/building-habits-key-lasting-behavior-change), [Frontiers in Public Health 2022 — Tiny Habits for Gratitude](https://www.frontiersin.org/journals/public-health/articles/10.3389/fpubh.2022.866992/full)

## 4. 재방문 — Spaced Repetition (Ebbinghaus → SuperMemo → Anki)

### (a) Forgetting Curve

- Ebbinghaus(1885): 새 정보의 **50%는 24시간 안에, 90%는 1주 안에 잊힌다** — 능동적 재학습이 없으면.
- 이 망각 곡선을 깨는 가장 검증된 방법이 spaced repetition.

### (b) 검증된 retention 향상

- 전통 학습 대비 **장기 retention 200–300% 증가**.
- *직접 만든* 플래시카드는 *기성 카드*보다 **43% 더 잘 기억된다** — generation effect가 spaced repetition 안에서도 작동.

### (c) 일일 실행 가이드

- 하루 **10–15분**이면 충분. 길이보다 *매일·같은 시간·같은 장소* 일관성이 결정적.
- 뇌가 그 시간을 예측하기 시작하면 의지력 부담이 사라진다.

### (d) 지식 노동자에게의 적용

- 단순 사실 암기뿐 아니라 *읽은 책의 핵심 주장, 미팅 결정사항, 디버깅 경험* 등을 spaced review에 태우면 누적이 일어난다.
- Andy Matuschak·Michael Nielsen 같은 PKM 실무자들이 SR을 사고 도구로 확장한 사례.

→ 출처: [Gwern — Spaced Repetition for Efficient Learning](https://gwern.net/spaced-repetition), [Cohorty Blog — Spaced Repetition Study Habit (Evidence-Based)](https://blog.cohorty.app/spaced-repetition-study-habit-evidence-based/), [ERIC EJ1241511 — Spaced Repetition: towards more effective learning in STEM (PDF)](https://files.eric.ed.gov/fulltext/EJ1241511.pdf)

## 5. 통합 — 네 층위가 어떻게 맞물리나

연구·방법론을 한 그림으로:

| 층위 | 다루는 문제 | 대표 답 |
|---|---|---|
| **노트의 구조** | "무엇을 어떻게 적나" | Zettelkasten — atomic + linked |
| **흐름** | "어떻게 처리·정착시키나" | CODE: Capture → Organize → Distill → Express |
| **습관** | "어떻게 매일 적는가" | Tiny Habits: anchor + 30초 + celebration |
| **재방문** | "어떻게 기억으로 만드나" | Spaced Repetition: 매일 10-15분 일관 |

각 층위는 독립적이지 않다:
- Zettelkasten의 atomic note가 있어야 spaced repetition이 가능 (덩어리 노트는 SR에 못 들어간다).
- CODE의 Express가 있어야 Distill에서 한 통찰이 generation effect로 박힌다.
- Tiny Habits가 없으면 Capture 자체가 작동하지 않는다.

## 6. 한 줄짜리 "시작 레시피" (연구가 합의하는 최소 단위)

(아래는 outbox 글을 쓸 때 본인이 *어느 것이 자기 상황에 맞는가* 골라보면 좋은 후보들. 그대로 박지 말 것 — generation effect를 살리려면 본인 손으로 다듬어야 한다.)

- **Recipe A** (Tiny Habits): "회의 끝나면 → 한 줄 결정 노트". 30초 이하. 회의실 나오는 행동이 anchor.
- **Recipe B** (Daily review): "잠자기 전 10분 → 오늘 배운 것 한 가지 안 보고 적기" (retrieval practice).
- **Recipe C** (Express-first): 새 자료 읽기 전, 자기 노트에 *기대하는 결론* 한 줄 먼저 쓰기. 읽고 나서 그 줄을 갱신.
- **Recipe D** (Atomic discipline): 노트 하나에 두 아이디어가 들어가는 순간 분리. 제목이 두 개의 명사구를 필요로 하면 두 개의 노트로.
- **Recipe E** (Spaced revisit): 매일 같은 시간 15분, 일주일 전 노트 다시 펼쳐 한 줄 코멘트 추가.
